## 1. 왜 하는가 (사업 목적)

- **고도몰** = 신규 쇼핑몰(오픈 목표 2026-12). 목적: 업계 1등인 **자사 메인쇼핑몰**의 힘으로 **경쟁 2·3등 쇼핑몰의 파이를 뺏어오는 것.** 2·3등에 올라온 상품은 기본적으로 자사에 다 있음.
- 메인몰 등록 상품 **수백~1000개**를 고도몰에 올릴 계획.
- 단, **메인몰의 섬네일·상세페이지 이미지는 그대로 재사용하지 않음** → 차별화된 상세페이지·섬네일 필요.
- 수백~1000개를 일일이 수작업 = 불가능한 노가다 → **AI를 총동원한 자동 대량 변환**이 목표.

## 2. 변환의 개념·방법·구조 (2026-07-07 테스트 성공)

테스트 대상 타입: **"생성기로 만든(또는 유사 폼)" 상세페이지.**
- 예시 상품: `[애널용품] 프로그레시브 비즈 애널 플러그(Progressive Biz Anal Plug) / 써뉴뉴` — 소스 엑셀 `docs/테스트(1).xlsx`.

### 변환 파이프라인
1. 메인몰 상세페이지는 결국 **세로로 긴 통이미지**지만, **제품이미지 구간 ↔ 텍스트 구간이 명확히 구분**돼 있음.
2. 통이미지에서 **제품이미지 부분만 잘라내/추려냄** (메인·서브·사이즈·패키지 등).
   - → *상품마다 누끼(보정) 이미지를 재수집·업로드하는 수고를 없애는 것이 이 프로젝트의 핵심 가치.*
3. 추출한 이미지를 **새 고도몰 전용 레이아웃 폼**에 맞춰 배치(메인/서브/사이즈/패키지 자리).
4. 각 이미지에 맞는 **텍스트를 AI로 새로 생성** — 하이브리드: **Qwen2.5-VL(영문 시각 묘사) → Super Gemma(한글 카피)**.
5. **메인이미지 기반 섬네일 자동생성**(기존 생성기 디폴트 기능)까지 고려 → 깔끔한 메인이미지 선택이 중요.
6. 소스 데이터 = **엑셀의 통이미지 URL/정보**. 출력 = **HTML 구조**(텍스트가 SEO에 유리 — 강한 장점).

### 산출물 (실물 증거)

- `test/` 폴더 12장 — 통이미지에서 쪼갠 섹션 이미지(메인이미지 / 핵심특징 / point-01×3 / point-02×3 / size / 동영상 / 옵션1·2 / 패키지).
- 사장님 평가(06:36): **"테스트는 대성공. 가능성이 확신으로 접어들고 있어."**

### 상품명 정리 규칙
| 요소 | 처리 |
|---|---|
| `C군` / `[완료]` | 작업자용 표식 → **제거** (완료 = 검수 완료 표시) |
| `[애널용품]` | 구매자용 대분류 → **유지 필수** (후에 고도몰 카테고리와 매칭) |

| 끝 숫자/`SNN` (6973…, SNN) | 물류센터 코드 → **구매자에게 숨김** |

### 소스 상세페이지 두 타입
- **(A) 생성기폼/유사 폼** — 제품이미지 + 텍스트가 깔끔히 구분 → 분할·재조립. **2026-07-07 성공.**
- **(B) 업체 통짜·광고성 이미지** — 폼이 아님 → 그대로 다운받아 사용 + 메인이미지·문구 한두 줄만 추출(섬네일용). 워터마크 일괄삭제는 검토중.

## 3. 생성기 3개 구조

1. **기존 메인쇼핑몰 전용 생성기** — 이미 이식됨 (= 현재 `src/components/detailBuilder`).
2. **고도몰 전용 생성기** (신규) — 기존을 복제. **속(기능)은 동일**: 이미지 업로드 · 직접 텍스트 입력 · AI 문구 생성 · 메인기반 섬네일 자동생성. **다른 건 오직 상세페이지 레이아웃 디자인** (사장님이 디자이너로서 커스텀 방향 주문 → 구조/코드는 Claude가 구현).
3. **상세페이지 리뉴얼(자동 대량변환)** — 레이아웃 폼은 2번과 **동일**.

> 즉 실질적으로 "기존 생성기 + 고도몰 생성기 1개"이며, 고도몰 생성기가 **신제품 수동등록**과 **자동변환 결과 개별수정**을 겸한다.

## 4. 고도몰 전용 생성기 필수 요건 (사장님 확정)

- 자동변환 결과가 **좌측 입력부의 섹션별 이미지 슬롯 + 텍스트 칸에 그대로 로드**되고, 우측 미리보기에 완성본이 보여야 함.
- 그래서 **신규 수동등록과 똑같이** 텍스트/이미지 수동수정·AI 문구 재생성이 가능해야 함.
  - 이유: 자동변환은 100% 완벽하지 않음 — **줄맞춤 문제·검수·문구 수정**이 남음. 사람이 한 번은 훑어야 함(AI 검수 가능성은 미정).
- **등록까지 담당**: 고도몰 API `write` 또는 엑셀 업로드. (오픈 전 일괄 등록 vs 오픈 후 개별 신상품 등록에 맞춰 유리한 방법 선택.)
