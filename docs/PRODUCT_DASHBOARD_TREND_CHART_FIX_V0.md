# Product Dashboard Trend Chart Fix v0

> **작성일**: 2026-06-26 · **브랜치**: `fix/product-dashboard-trend-chart-v0`
> **코드**: `src/services/productDashboardTrendBuckets.ts`(신규 pure helper) + `src/components/ProductTeamDashboard.tsx` · **smoke**: `scripts/smoke-product-dashboard-trend-buckets.mjs`(10/10)

## 1. 문제 상황 (눈검수)
1. 월별 그래프 x축 라벨이 `7월, 9월, 11월`처럼 **띄엄띄엄** 표시(`labelStep=ceil(n/8)`).
2. 데이터 있는 기간만 버킷 생성 → **빈 월이 누락**되어 x축이 끊김(연속 아님).
3. 직접 기간 선택 시 단위별 버킷 수/라벨이 선택 기간과 정확히 안 맞음.
4. "가상 현재 재고 N개" KPI는 운영 의사결정에 유용성 낮음.

## 2. 수정한 bucket 정책 (`buildTrendBuckets`)
- **선택 기간(effStart~effEnd)으로 먼저 필터 → 그 안만 단위 버킷팅**. 기간 밖 데이터는 **절대 미반영**.
- **연속 버킷**: 선택 기간을 단위별로 빈 구간(0) 포함해 채운다 → x축이 끊기지 않음.
- 막대(주문수)·꺾은선(매출)은 **같은 버킷 리스트** 공유(함수 결과 하나).
- 데이터 소스/RevenueOrder 변경 없음. 순수 helper로 분리(과한 리팩토링 없음).

| 단위 | 버킷 생성 |
|---|---|
| month | start월~end월 연속(YYYY-MM), 라벨 `M월` |
| week | **start 기준 7일 윈도우**(7/2~7/8, 7/9~7/15 …), 마지막은 end로 클램프 |
| day | start~end 일별 연속, 라벨 `M/D` |

## 3. x축 라벨 정책 (`labelStepFor` — 버킷 유지, 라벨만 축약)
- month: ≤18 → 전부 표시, 초과 → `ceil(n/12)`
- week: ≤20 → 전부, 초과 → `ceil(n/16)`
- day: ≤14 → 전부, 초과 → `max(2, ceil(n/15))` (2~3일 간격)
- **데이터 포인트/막대는 항상 유지**, tooltip은 전체 표시.

## 4. 직접 기간 처리 규칙
- `2025-07-02~2025-12-26` + 월별 → 7,8,9,10,11,12 = **6 버킷**(부분 월 포함).
- `2025-07-02~2025-07-31` + 주간 → 7/2~7/8 … 7/30~7/31 = **5 버킷**(기간 밖 월 라벨 없음).
- `2025-07-02~2025-07-31` + 일별 → **30 버킷**, 라벨만 축약.
- 날짜는 native `<input type="date">` 값을 그대로 사용(파싱 연도 변형 없음). 로컬 날짜 구성으로 UTC 시프트 방지.

## 5. KPI 카드 변경
- 기존: `🏬 가상 현재 재고 655개 · 관리 상품 13개`
- 변경: `📦 재고 위험 상품 N개 · 주의 M개 · 관리 상품 13개 · 최저 <상품명> Q개`
- 이유: 전체 재고 합보다 **위험/주의 상품 수 + 최저재고**가 운영에 직접 유용. 기존 stockImpact 재사용(danger≤20 / warn 21~40 / 최저재고 상품).
- 카드 1개의 제목/값/보조정보만 변경(레이아웃·구조 무변경).

## 6. 매출 구성 도넛 동기화
- 도넛은 `categoryData`(= `ordersFiltered` 기준)로 계산 → **매출추이와 같은 `effStart/effEnd` 기간 필터를 공유**(기존부터 동기화됨, 변경 불필요). aggregation unit(월/주/일)은 도넛에 영향 없음(기간 필터만).

## 7. 검증 시나리오 (smoke로 단위 검증)
- 기본 월별 12개월 → 12 버킷(7월~6월 전부 라벨) ✅
- 2025-07~2025-12 월별 → 6 버킷 ✅
- 2025-07-02~07-31 주간 → 5 버킷(기간 밖 미표시) ✅
- 2025-07-02~07-31 일별 → 30 버킷 ✅
- 기간 밖 주문 제외 + 버킷 범위 내 + 막대/꺾은선 동일 count ✅

## 8. 남은 이슈
- 브라우저 눈검수(실 대시보드): 라벨 겹침 시 폰트/회전 미세조정 여지.
- 매우 긴 일별 범위(수백 일) 시 라벨 축약은 동작하나 데이터 포인트 다수 → 필요 시 day 상한 UX 검토.
- 재고 위험 카드의 "최저재고" 상품 클릭 → 상세 드릴다운 연결(후속).
