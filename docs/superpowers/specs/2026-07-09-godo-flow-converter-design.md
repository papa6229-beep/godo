# 단순형(Flow) 변환기 렌더러 — 설계 (2026-07-09)

## 배경
메인몰 상세페이지 5개 타입 → 출력구조 2개로 수렴:
- **A. 섹션형**(기존 고도몰 생성기) = 타입1(기본형)
- **B. 단순 플로우형**(신규) = 타입2~5(단순형): 상단 텍스트 + 통이미지 세로 스택

이번 조각 = **B(단순 플로우) 렌더러 신설**. 수동 입력으로 먼저 "그릇" 완성 → 이후 엑셀 프리필/분할/VLM이 자동 채움.

## 최우선 제약: 기존 고도몰 생성기 무손상
`PreviewGodo.tsx`·`Editor.tsx`(섹션형)는 **한 줄도 수정 안 함**. 변환기는 완전 분리된 신규 컴포넌트 + 공용 인프라 추가분기(additive)로만.

## 구조 (결정: 원본 충실 스택형, 사장님 확정)
- 본문 = `[상단 텍스트]` → `[통이미지 세로 스택]` → `[footer]`. **메인/스펙/포인트/사이즈 섹션 없음.**
- **섬네일은 본문과 분리**: `mainImage` 필드를 flow 모드에선 "섬네일 소스"로만 사용(본문 미표시) → 기존 섬네일 4종 자동생성기에 투입. 마땅한 이미지 없으면 비워도 됨.
- 옵션(타입5) = 기존 `options[]` 재사용(옵션명+이미지 → 고도몰 옵션 등록). 지금은 수동.
- 재사용: 테마·브랜드(footer)·워터마크·이미지 테두리(IMG_BORDER)·간격 드래그·export·섬네일.

## 변경 목록
1. `types.ts` — 추가(옵셔널): `flowHeaderText?: string`, `flowImages?: string[]`. (기존 필드 불변)
2. **신규** `components/PreviewGodoFlow.tsx` — flow 렌더러(격리). 헤더텍스트 + flowImages 스택(풀폭·테두리·워터마크·간격) + footer.
3. **신규** `components/EditorFlow.tsx` — flow 에디터(격리). 상품명·브랜드·테마·헤더텍스트·**순서형 이미지 스택 업로더**·섬네일소스·워터마크·옵션.
4. `DetailPageBuilder.tsx` — `layoutMode: 'godoFlow'` 추가분기: 미리보기→PreviewGodoFlow, 에디터→EditorFlow, 섬네일 layoutMode='godo'로. 기존 godo/bananamall 경로 불변.
5. `DesignTeamDashboard.tsx` — 3번째 진입 버튼 "🔄 변환기(단순형)" + builderMode 타입 확장.
