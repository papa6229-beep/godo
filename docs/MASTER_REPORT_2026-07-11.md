## PART 2. 오늘 실패/이탈한 것 (오후) — 왜 산으로 갔나

### 2-1. 인앱 파이프라인 이식 시도 (commits `1ac3295`~`6a54e3a`, **되돌림**)
v3에 만족한 뒤 "이 지능을 앱에 이식"하겠다며 여러 서비스/UI를 만듦: `godoBasicConvert.ts`(조립), `classifyImageRoles`(밴드 역할 VLM 분류), `generateCaptionsForImages`(이미지별 캡션 생성), `lineBreakForLayout`, `computePackageLayout`, `runBasicConversion`, Editor의 `BasicConvertPanel`. dev에서 end-to-end 실행까지 성공(핑거위글 8컷 업로드→변환→렌더).

### 2-2. 치명적 문제 = 속도 + 개념 오류
- **8컷 변환에 6분.** 이후 배치화로 ~57초로 줄였지만 여전히 느림. **1000+ 상품엔 불가.**
- **근본 원인**: 나는 **"이미지 보고 글을 새로 생성"**(VLM 묘사→Gemma 작문)을 시켰다. 이건 07-08 C-5(**팩트 유지, 표현만 변주=라이트 리라이트**)와 C-3(**원본 텍스트 큐레이션·재배치**)를 정면 위반. **원본 글을 살짝 비트는 게 아니라 없는 글을 지어낸 것.**
- 단순형(옵션닛포리)이 0.5초에 되는 건 **엑셀에 글이 이미 분리돼 있어 → 라이트 리라이트만** 하기 때문. 나는 그 빠른 길을 버리고 느린 생성을 기본으로 깔았다.

### 2-3. 내가 저지른 프로세스 실패 (기록해서 재발 방지)
1. **확인 안 하고 단정(추측/할루시)**: "핑거위글 baked라 글 없음"(→ 실제 엑셀엔 상단 상품명·요약 텍스트 있음), "성인이라 클라우드 거부 → 버셀 안 됨"(→ 코드에 그 가정이 있긴 하나 항상 참 아님; 나 Claude=클라우드가 거부 없이 처리) 등. 사장님이 매번 바로잡아야 했다.
2. **왔다갔다(flip-flop)**: 위를 말했다가, 지적받자 확 뒤집었다가, 코드 확인하니 반은 맞고 반은 틀림. 확인부터 했어야.
3. **위화감 부재**: 상품 1개에 6분 걸리면 **내가 먼저 "이거 너무 느린데요" 하고 멈췄어야** 했다(1000+ 스케일 자각). 사장님이 지적할 때까지 몰랐다.
4. **과잉복잡화**: v3가 이미 답을 보여줬는데, 잘 되던 걸 더 복잡하고 나쁘게(느리고 글 없이) 만들었다.
- **교훈(반드시 지킬 것)**: ①단정 전 코드/파일 열어 확인 ②속도가 대전제(1000+) — 느리면 스스로 멈추고 재검토 ③잘 되던 방식(단순형 라이트 리라이트) 재사용, 새로 짓지 말 것 ④말 한마디가 비전문가 사장님께 오해를 낳고 지시가 꼬이고 작업이 산으로 감 → 정확·간결.

## PART 4. 개념·기술스택 재정립 (사장님이 바로잡아준 정본 — 다시 설명 안 해도 되게)

### 4-1. 상세페이지 타입과 텍스트 소재의 진짜 구분
- **닛포리형(단순형, typed)**: 엑셀 상세설명 HTML에 **이미지와 텍스트가 분리돼** 있음(`detailBlocks` = 이미지+텍스트 페어). → **읽을 필요 없음.** 원본 텍스트 라이트 리라이트 + 스택. **0.5초, 버셀 OK(클라우드면).**
- **트리니티형(단순형)·기본형 = 통이미지(baked)**: 상품명·요약 정도만 엑셀에 텍스트로 있고, **각 섹션 설명·스펙은 통이미지 픽셀에 박혀** 있음. → **통이미지에서 ①이미지 추출 + ②텍스트 추출(박힌 글 읽어내기)** 해서 따로 넣는 **별도 과정 필요.** 그 다음은 동일(라이트 리라이트 + 배치).
- **기본형과 단순형의 유일한 차이 = 출력 레이아웃.** 기본형=고도몰 섹션 슬롯(메인/feature/point/size/package)에 배치, 단순형=세로 스택. **AI 역할은 동일**(원본 글 살짝 비틀기). 성인 파트도 동일.

### 4-2. AI 연결 사실 (코드 확인 — 추측 아님)
- **텍스트 리라이트**(`rewriteFlowCaptions`): `resolveAgentBrain('design')` = **설정 가능한 "디자인 AI"**. `DesignTeamDashboard`가 **기본값을 로컬 Super Gemma로 고정**(주석: 성인이라 클라우드 거부 가정). 클라우드(Claude/GPT/Gemini+키)로 바꾸면 **버셀에서도 됨**. `isBrainConnected`: 클라우드=키만 있으면 어디서나 / 로컬=dev·localhost만.
- **비전(이미지에서 글 읽기)**(`getChatCompletion`/`lmsConnector`): **`127.0.0.1:1234`(로컬 LM Studio)에 하드코딩** → **dev 전용.** 버셀에서 통이미지 글을 읽으려면 **클라우드 비전(성인 거부 미검증)** 또는 **OCR(글자만 인식, 성인 무관, 브라우저에서 돎=버셀 OK, 속도 빠름)** 필요.
- **빨간 볼드 강조**: `##키워드##` → `renderHighlight`(PreviewGodoFlow) = **클라이언트 CSS**(볼드800 + 테마색 #E11D48). **AI 아님, 버셀 포함 어디서나 됨.** AI는 `##` 마커만 삽입.
- **정리**: 어제 사장님이 본 "옵션닛포리 문구 살짝 비틀기 + 빨간 볼드" = **dev + 로컬 Super Gemma**였음. 지금 버셀이 원본 그대로인 이유 = 리라이트(로컬 AI)가 버셀에서 미연결이라서.

### 4-3. 오늘 dev 실증(성공) — 옵션닛포리 라이트 리라이트 ✅
`test/옵션닛포리.xlsx` 업로드→닛포리형(원본 설명 18개 프리필)→🤖 AI 리라이트 → **문구 살짝 비틀림 + 중요 포인트 빨간 볼드(#E11D48)** 정상 렌더(`test/nippori_red_caption.jpeg`). **단, 18개에 ~150초**(Super Gemma 26B, text-only 배치 CHUNK=10 병렬). → **이 속도가 월요일 첫 과제.**
