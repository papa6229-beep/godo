## PART 2. 오늘의 할루시 — 자체점검 (사장님이 캐물어 도출)

### 무슨 일
사각 금색 프레임을 ①검출 로직에서 빠뜨리고 ②검수에서 못 보고 ③"둥근 모서리"라 잘못 묘사. 사장님이 "각진 네모"라 교정.

### 원인(자체점검 결론 — 이건 가설, 행동으로 검증됨)
1. **물려받은 잘못된 구조 모델**: 강종세션 로그+메모리+보고서에서 "통이미지=가로 여백/금색선 스택"을 그대로 이어받고 **원본을 새 눈으로 재관측 안 함.** → 코드가 가로(1D)만 처리.
2. **관측 도구가 1D**: 행 스캔은 세로 프레임을 원리적으로 못 봄. 그 수치가 깨끗하니 "깨끗하다" 믿음.
3. **확증 편향**: 크롭을 "됐나"로 봄, "뭐가 남았나"로 안 봄.
4. **미세 디테일은 확대 없이 흘긋 → 예상으로 메꿈.**

### 사장님이 준 통찰(중요)
매일 마스터보고서가 커지고 **결론까지 씹어서 담을수록**, 나는 원본 대신 그 결론 위에서 놀아 추측 영역이 늘어남. **정보량보다 "결론을 사실처럼 담은 것"이 독.** → 보고서는 **포인터+결정사실+검증된것(+재확인법)+열린질문**만. 결론/모델은 "가설, 재확인 필요"로 표시.

### 신뢰 규칙 (사장님과 합의)
- 내 **검증 불가능한 주장**(내부 이유·"검증했다")은 의심.
- **검증 가능한 것**(좌표·픽셀·확대컷)만, 사장님이 원본과 대조해 판단.
- 오늘 실증: BR 코너 확대컷 = 검증 가능 → 사장님이 직접 확인 → **내가 틀리고 사장님이 맞음이 근거로 증명됨.**

### 3-3. ⬅ 사장님 결정 필요 (내가 A/B 표로 낼 것)
