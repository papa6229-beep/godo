## PART 2. 오늘 잡은 철학 (사장님 강한 교정 — 가장 중요)

> **이건 공식·매뉴얼이 아니라 "참고자료·고려사항·변수"다. AI가 그걸 참고해 지능적으로 판단하는 게 핵심.** 단순하면 그대로, 이상하면 융통성. "그럴거면 파이썬을 쓰면 돼."

- **기준은 항상 고도몰 생성기 슬롯(고정 그릇).** 기본형은 그 그릇을 채울 **재료**. 분할은 고도몰 레이아웃이 이미 해놨다(point01 영역이 곧 슬롯) — 기본형 pill을 감지하겠다는 발상 자체가 틀림(기계적 사고).
- **고도몰에 있는 자리** → 기본형에서 맞는 걸 골라 넣고, **없는 자리(영문 대제목 등)** → 그냥 버리거나, 쓸 만한 제품 설명이면 다른 칸에 갈무리. 능동적 판단.
- 동작 원리는 단순: **이미지 박고 그에 맞는 특징 텍스트, 반대로 텍스트 박고 맞는 이미지.** 애매하면 기본형 이미지 순서대로 먼저 박고, 텍스트는 원본 보고 고도몰 칸 의미에 맞춰 채움.
- **선택존**: 이미지+설명 페어가 많으면(point03,04…) godo 섹션을 늘릴지, 맥락 맞는 이미지들을 세로로 길게 합칠지 — AI가 판단. 설명 없이 마케팅 통이미지만 이어붙인 상세는 그대로 스택.
- ※ 이 톤을 못 잡아 오늘 두 번 헛질문함(상단요약을 어디 넣나 / 분할 주도권 결정론 vs VLM) → 둘 다 "그릇은 이미 정해져 있는데 재료를 기계로 스캔하려던" 실수. 교정 완료.

**핵심 결론(결정론적으로 처리 가능한 것):**
- **기본형 소스 = 통짜 1장이 아니라 이미지 URL 리스트**(goodsm jpg 여러 장 + gif). 제가 "디테일컷(연출)"로 착각했던 게 실은 `_1.gif` 움짤.

- **변환 대상 = `files/goodsm/` jpg** (순서 = 상세 위→아래). 이 안이 픽셀 텍스트라 **여백 감지로 조각 분할**하는 게 실제 작업.
- 참고: 목록이미지(`files/goods/…gif`)는 바나나몰 로고 박혀서 섬네일로 못 씀 → 섬네일은 상세 누끼컷에서 뽑음. (07-09 실측 재확인)

## PART 8. 재개 시 주의할 기술 사실

1. **도구 환경**: sharp/canvas 없음, **ImageMagick 없음**(`convert`는 Windows NTFS 유틸이니 쓰지 말 것), **Python 3.10 + Pillow 11.2 + numpy 2.2 사용**. 이미지 슬라이싱=Pillow.
2. **Playwright는 `file://` 차단** → 로컬 `python -m http.server <port> --bind 127.0.0.1` 띄우고 `http://127.0.0.1:...`로 접근. 끝나면 `pkill -f http.server`.
3. **PreviewGodo 레이아웃 상수**: 컨테이너 width 800·px-50(내용 700)·Pretendard. 섹션 헤딩 72px black. 스펙 2행(1행 타입/재질/치수, 2행 무게/전원, 각 열 폭 145). 무게 있으면 SIZE 섹션 자동. 마퀴=영문명 6회 반복(KEY FEATURE 아래·SIZE 위). GODO_BRAND=footer/섬네일 단일소스(아직 플레이스홀더 'GODO MALL').
4. **간격 시스템**: `godoSpacing{section,element,heading}`(종류별 스칼라) + `godoGaps{위치id:px}`(위치별 오버라이드). 미리보기 GapBar/SectionGap 드래그로 조절, 임시저장으로 고정. 내일 A-1은 이 **기본값** 튜닝.
5. **Tailwind preflight OFF** → 테두리는 인라인 `border`(유틸 무효). react-rnd 중첩 오프셋 버그 → 커스텀 absolute 드래그(feature 이미지).
6. **CORS**: 실제 CDN 이미지는 표시(img src)만 OK, canvas 픽셀작업(export/워터마크)은 서버 fetch base64 필요. PoC는 로컬 파일이라 무관.
7. **상품명 클렌징**: `[대분류]` 유지 · 물류코드/벤더약자(끝의 `(SW1064-2)(SWL)(TJ)`류) 제거 · 브랜드/영문 유지.
